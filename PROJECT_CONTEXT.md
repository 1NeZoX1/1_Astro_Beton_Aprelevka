# Project Context

Проект: SEO-сайт-каталог на Astro для ООО “ЛИДЕР БЕТОН ГРУПП”.

Компания продает бетон и строительные смеси с доставкой по Москве и Московской области.

## Технологии

- Astro
- TypeScript
- статическая генерация
- products.json как источник товаров
- schema.org
- sitemap
- robots.txt

## Данные

- products.json содержит товары с полями category/name или category/title/price
- slug, categorySlug, seoTitle, seoDescription и описания генерируются в catalog.ts
- арматуры в данных быть не должно
- ошибочные позиции “12” и “Бетон с ПМД+A64:G68” нужно удалить или фильтровать

## Каталог

Верхние крупные категории:

- Бетон
- Бетон с ПМД
- Фибробетон
- Керамзитобетон
- Сухой бетон
- Растворы и ЦПС
- Аренда техники
- Другое

Маршруты:

- /catalog/ — страница выбора крупных категорий.
- /catalog/[categorySlug]/ — страница выбранной категории с товарами.
- /catalog/[categorySlug]/[productSlug]/ — страница товара.

На главной не нужно выводить все 204 товара. На главной в блоке цен должны быть карточки крупных категорий с переходом в каталог.

## Фильтры и сортировка

- фильтры товаров работают внутри блока “Марки и цены” на странице категории
- фильтры не являются отдельными категориями сайта и не должны создавать новые маршруты
- ПМД-фильтр должен быть только на /catalog/beton-s-pmd/
- внутренние фильтры есть для:
  - Бетон с ПМД — Все, ПМД-5, ПМД-10, ПМД-15
  - Фибробетон — по марке M
  - Керамзитобетон — по классу B, включая варианты B12,5/B12.5 и B7,5/B7.5
  - Сухой бетон — по марке M
  - Растворы и ЦПС — по типу продукта
  - Аренда техники — по типу услуги
- фильтр не показывается, если в категории есть только один полезный вариант фильтра
- сортировка товаров: сначала марка M по убыванию, затем цена по убыванию, затем название
- если марки M нет, сортировать по цене по убыванию, затем по названию
- в “Бетон с ПМД” сначала группировка по ПМД, внутри группы сортировка по M/цене/названию
- карточки товаров должны оставаться компактными: без больших изображений, с бейджем категории/подгруппы, бейджем марки M, характеристиками B/П/F/W, ценой и кнопками

## Проверка

- не запускать `npm run build` автоматически после каждого изменения, чтобы не тратить лишние токены
- запускать сборку только когда это действительно нужно: перед коммитом/пушем, по явной просьбе, либо после рискованных изменений маршрутов, генерации каталога, schema.org или sitemap, и даже в таком случае спрашивать разрешения запуска сборки
- в текущем окружении PowerShell может блокировать `npm run build` через `npm.ps1`; тогда использовать эквивалентную команду:

```powershell
npm.cmd run build
```
